## 仍待处理 / 已知问题

### P7 必须完成

1. **共享 Docker 网络**：NyaaCLI 和 nyaadiypc-mcp 需要同一 Docker 网络才能通过容器名互访。当前各自 compose 创建独立 default 网络。需要在 macmini 上建共享网络（如 `nyaanet`），两个 compose 文件都加入该网络。
2. **NyaaCLI 重建部署**：NyaaCLI 的 mcp.json/Dockerfile/entrypoint.sh 已改好但**尚未重建推送到 macmini**。需要：
   - NyaaCLI 仓库 `python rebuild.py` 构建推送
   - macmini 侧 NyaaCLI `python3 restart.py` 部署
   - 首次启动后 `/workspace/.claude/mcp.json` 自动就位（已存在于 workspace 则需手动覆盖或删除后重建）
3. **定时爬虫任务**：macmini 容器内配置 cron 定时执行 `python -m app.crawler.run --category all`
4. **全链路 E2E**：NyaaCLI 容器内 claude 实际调用 diy-pc-mcp 的 build_pc 走通
5. **真实数据校准**：ZOL 爬虫填充真实硬件数据后，校准 demand_map 中的天梯分阈值

### 可选改进

- **外部访问 Host 校验**：当前 Starlette 拒绝非 localhost 的外部 MCP 请求（安全特性），内部 Docker 网络互访正常。如需外部调试，可在 server.py 配置 `allowed_hosts`。
- **search_hardware 扩展**：可增加更多过滤维度（接口类型、板型、功率等）
- **W2 内存频率建议**：P4 遗留占位，待补

## 已知问题 / 待处理

- **无实际硬件数据**：P5 测试用 synthetic test data，完整 build_pc 效果依赖 P2 爬虫填充的真实 ZOL 数据。当前 `demand_map` 播种的天梯分阈值（80-220）是基于经验的估计值，实际 ZOL 天梯分标尺可能不同，需在真实数据就位后校准。
- **GPU selection 可能打不中**：`select_gpu` 先按 `rec_tier` 最便宜选，若候选池为空降 `min_tier`。没有真实 tier_score 数据时，此逻辑退化为纯预算选件。
- **收敛可能不达标**：极窄预算区间（如 8000-8200）可能选不到满足 tier 门槛的 GPU/CPU，此时方案标 `constraint_violation` 而非硬凑不兼容件。
- **search_hardware 简化**：当前仅支持 category/keyword/price 过滤，后续可扩展更多过滤维度（接口类型、板型、功率等）。
- **W2 仍占位**（P4 遗留），内存频率建议逻辑待补。

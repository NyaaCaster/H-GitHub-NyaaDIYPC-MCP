# 02 数据模型详细设计（SQLite Schema）

> 对应 SSOT 第 2 节。执行期以本文件建表，字段/类型/约束不再决策。
> compat 采用「结构化列 + specs_json 原文」双写：结构化列供 SQL 过滤与兼容校验，specs_json 保留全量原始规格供展示与回溯。

---

## 1. 表 `hardware`（硬件件目录）

```sql
CREATE TABLE hardware (
  pro_id        TEXT PRIMARY KEY,        -- ZOL proId，全局唯一，天梯/价格 join 键
  category      TEXT NOT NULL,           -- cpu|mainboard|memory|gpu|hdd|ssd|psu|cooler|case
  manu_id       TEXT,                    -- 品牌 id（relmanu）
  brand         TEXT,                    -- 品牌名（从型号或深参提取）
  model         TEXT NOT NULL,           -- 型号全名
  -- 价格（分位存 int RMB 元；无则 NULL）
  price_jd      INTEGER,                 -- 京东价（预算判定主基准）
  price_show    INTEGER,                 -- 攒机页展示价
  price_min     INTEGER,                 -- 多商家最低价
  price_rt      INTEGER,                 -- best-price 实时补价（淘宝/京东，仅参考）
  price_rt_src  TEXT,                    -- 实时价来源标签
  price_rt_at   TEXT,                    -- 实时价抓取时间 ISO8601
  -- 性能
  tier_score    REAL,                    -- 天梯分（cpu=game维度, gpu=显卡分）；同类可比，跨类不可比
  tier_rank     INTEGER,                 -- 天梯排名
  -- 其他
  popularity    INTEGER DEFAULT 0,       -- 本月已选用人数（热度）
  specs_json    TEXT,                    -- 全量规格 JSON（内嵌+深参合并原文）
  active        INTEGER DEFAULT 1,       -- 1在售 0停售
  deep_fetched_at TEXT,                  -- 深参最后抓取时间；NULL=未抓深参
  fetched_at    TEXT NOT NULL            -- 本条最后刷新时间 ISO8601
);
CREATE INDEX idx_hw_cat        ON hardware(category, active);
CREATE INDEX idx_hw_cat_price  ON hardware(category, price_jd);
CREATE INDEX idx_hw_cat_tier   ON hardware(category, tier_score);
```

## 2. 表 `compat`（兼容归一字段，1:1 hardware）

拆独立表：字段随品类而异，避免 hardware 巨宽。按 `pro_id` 外键 1:1。

```sql
CREATE TABLE compat (
  pro_id        TEXT PRIMARY KEY REFERENCES hardware(pro_id),
  category      TEXT NOT NULL,
  -- CPU
  socket        TEXT,        -- LGA1700 / AM5 ...（归一后）
  tdp_w         INTEGER,     -- 热设计功耗
  mem_type      TEXT,        -- DDR5 / DDR4
  igpu          INTEGER,     -- 有无核显 1/0（选配无独显时用）
  -- 主板
  form_factor   TEXT,        -- ATX / M-ATX / ITX（主板板型）
  mem_slots     INTEGER,     -- 内存插槽数
  mb_chipset    TEXT,        -- Z790 / B760 / X670 ...
  m2_slots      INTEGER,     -- M.2 接口数
  -- 内存
  mem_capacity_gb INTEGER,   -- 单套总容量
  mem_freq      INTEGER,     -- 频率 MHz
  -- 显卡
  vram_gb       INTEGER,
  gpu_len_mm    INTEGER,     -- 显卡长度
  gpu_power_pin TEXT,        -- 电源接口（8pin*2 / 16pin ...）
  gpu_rec_psu_w INTEGER,     -- 建议电源功率
  -- 存储
  interface     TEXT,        -- M.2 NVMe / SATA ...
  ss_form       TEXT,        -- M.2 2280 / 2.5in ...
  capacity_gb   INTEGER,
  size_inch     REAL,        -- HDD 3.5/2.5
  -- 电源
  rated_w       INTEGER,     -- 额定功率
  modular       TEXT,        -- 全模/半模/非模
  cert          TEXT,        -- 80PLUS 认证等级
  -- 散热
  cooler_type   TEXT,        -- 风冷/水冷
  cooler_h_mm   INTEGER,     -- 散热器高度
  cooler_sockets TEXT,       -- 支持插槽 JSON 数组字符串
  -- 机箱
  case_ff       TEXT,        -- 支持板型 JSON 数组字符串（ATX/M-ATX/ITX）
  max_gpu_len_mm INTEGER,
  max_cooler_h_mm INTEGER,
  radiator_support TEXT      -- 支持水冷冷排位 JSON
);
CREATE INDEX idx_compat_cat ON compat(category);
```

> 缺失字段一律置 NULL；兼容校验对 NULL 走「未知→放行+告警」（见 04）。

## 3. 表 `perf_tier`（天梯原始，供刷新对齐与展示）

```sql
CREATE TABLE perf_tier (
  pro_id     TEXT NOT NULL,
  kind       TEXT NOT NULL,      -- cpu | gpu
  dimension  TEXT NOT NULL,      -- cpu: game/colligate/singleCore/multiCore; gpu: default
  model      TEXT,
  score      REAL,
  rank       INTEGER,
  ratio      TEXT,               -- 领先百分比原文
  firm       TEXT,               -- cpu: 1 Intel 2 AMD
  fetched_at TEXT NOT NULL,
  PRIMARY KEY (pro_id, kind, dimension)
);
```

`hardware.tier_score` 从 `perf_tier`（cpu 取 game、gpu 取 default）回填，方便直接查询。

## 4. 表 `demand_map`（需求映射，D5）

详见 05 设计，此处仅列结构：

```sql
CREATE TABLE demand_map (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  game         TEXT NOT NULL,        -- 游戏/场景标识（归一小写）
  aliases      TEXT,                 -- 别名 JSON（"生化危机9","re9","biohazard9"）
  resolution   TEXT NOT NULL,        -- 1080p|2k|4k
  quality      TEXT NOT NULL,        -- low|medium|high|ultra
  fps_target   INTEGER,              -- 目标帧
  min_cpu_tier REAL,                 -- 所需最低 CPU game 分
  rec_cpu_tier REAL,                 -- 推荐 CPU game 分
  min_gpu_tier REAL,                 -- 所需最低 GPU 分
  rec_gpu_tier REAL,                 -- 推荐 GPU 分
  min_vram_gb  INTEGER,
  min_ram_gb   INTEGER,
  note         TEXT
);
CREATE INDEX idx_demand_game ON demand_map(game, resolution, quality);
```

## 6. 数据完整性约束（执行期铁律）

1. 任何写库前**先复制 `.db` 带时间戳备份**（SSOT §7）。
2. `price_jd` 为预算判定唯一主基准；`price_rt` 仅在 build_pc 展示时附注，不参与 in_budget 判定。
3. `tier_score` 同类可比、跨类不可比，需求映射与搭配算法严禁 CPU/GPU 分值直接相加比较。
4. 刷新用事务；停售件 `active=0` 不物删。
5. `specs_json` 永远保留原文，结构化列仅作过滤/校验的派生视图。

# 03 价格与对齐详细设计（D4 + 天梯/型号匹配）

> 对应 SSOT D4 与审计整改 2。执行期照此实现，价格来源优先级、补价过滤、匹配算法均已钉死。

---

## 1. 价格来源与优先级（D4 定案）

| 来源 | 字段 | 角色 | 参与预算判定 |
|------|------|------|:---:|
| ZOL 京东价 | `price_jd` | **主基准** | ✅ 唯一 |
| ZOL 展示价 | `price_show` | 兜底（jd 缺时候补） | 仅兜底 |
| ZOL 多商家最低 | `price_min` | 参考下限 | ❌ |
| best-price 实时 | `price_rt` | 补淘宝价、时效校正 | ❌（仅展示附注） |

### 1.1 单件生效价 `effective_price` 决策

```
effective_price =
    price_jd            if price_jd 有效
    else price_show     if price_show 有效
    else price_min      if price_min 有效
    else None           # 无价 → 该件不可进方案
```

> `in_budget` 判定、`total` 求和**只用 effective_price**。`price_rt` 永不进入求和，只在方案条目上以「实时参考：淘宝￥xxx（{time}）」形式附注。

## 2. best-price 补价规则（审计整改 2 强制过滤层）

### 2.1 何时补

- **仅对已进入候选方案的件**补实时价（通常 ≤ 20 件/次），不整库补。
- 补价是**旁路增强**，失败/超时不阻断出方案。

### 2.2 强制过滤层（防串货/整机混入污染）

调用 `best-price` 返回后，逐条过滤，**任一不满足则丢弃该实时价**：

```
1. 关键词命中：结果标题须含目标件品牌 + 型号核心 token（归一后匹配，见 §3）
2. 整机排除：标题含「整机/主机/台式机/组装机/准系统/套装/全套」→ 丢弃
3. 串货/异常价排除：price_rt < effective_price * 0.5 或 > effective_price * 2.0 → 丢弃（判为串货/配件/错配）
4. 平台白名单：仅接受淘宝/天猫/京东来源；其他来源仅记录不采用
5. 淘宝价定位：淘宝命中价写 price_rt + price_rt_src="taobao"；京东实时命中可校正但仍以 ZOL price_jd 为预算基准
```

过滤后无有效结果 → `price_rt=NULL`，方案照常用 `effective_price`。

### 2.3 时效

`price_rt_at` 记录抓取时间；build_pc 输出 `priced_at`（方案定价时刻）。实时价超过 `RT_PRICE_TTL`（默认 6h）视为过期，展示时标「（价格可能已变动）」。

## 3. 型号归一与匹配算法（天梯对齐 + best 补价共用）

### 3.1 归一函数 `normalize_model(name)`

```
1. 转小写、全角转半角
2. 去括号补充：（游戏）（超频版）(OC) 【】 等 → 去除
3. 去营销后缀 token：vulcan / gaming / oc / trio / suprim / 魔龙 / 万图师 ... （维护一份后缀词表 suffix_stopwords）
4. 统一空白、去连字符差异：ryzen 9 9950x3d / ryzen9-9950x3d → 统一
5. 抽核心型号 token：
   - CPU: 抽 (i[3579]-\d{4,5}[a-z]*) | (ryzen \d \d{3,4}[a-z0-9]*) | (r[3579] \d{3,4}[a-z]*)
   - GPU: 抽 (rtx ?\d{4}[a-z ]*(ti|super)?) | (rx ?\d{4}[a-z]* ?(xt)?) | (arc [ab]\d{3})
```

### 3.2 匹配优先级

```
1. proId 精确 join（天梯 proId == hardware pro_id）—— 首选，命中即用
2. 归一核心型号 token 完全相等
3. 归一后 token 集合包含关系 + 品牌一致
4. 以上都不中 → 不匹配（tier_score 置 NULL，不瞎配）
```

> 严禁模糊到「9950x3d 配到 9950x」这类错配。3 级匹配要求核心数字型号完全一致，仅允许营销后缀差异。

### 3.3 补价匹配（best-price 场景）

复用 3.1 归一 + 3.2 的 2/3 级判断；再叠加 §2.2 的整机排除与异常价排除。

## 2. GetGoods 接口契约

### 2.1 请求

```
GET https://zj.zol.com.cn/index.php?c=Ajax_ParamResponse&a=GetGoods&subcateId={ID}&type=1&page={N}
Headers:
  Referer: https://zj.zol.com.cn/          # 必需，否则跳 service.zol.com.cn/checking
  User-Agent: <常规桌面浏览器 UA>
```

### 2.2 响应（5 字段，实测确认）

```json
{ "maxPage": "25", "allNum": "220", "page": "1", "data": "<GBK HTML 片段>", "sta": <状态> }
```

- 响应体按 **GBK 解码**再 `json.loads`：`resp.content.decode('gbk')`。
- `data` 是商品列表 HTML；分页 `1..int(maxPage)`。

### 2.3 9 品类 subcateId（务必以此为准，未知 ID 会 fallback 到 CPU）

| 品类 | category 枚举 | subcateId | 详情路径前缀 |

| 内存 | `memory` | 3 | /memory/ |

| 机械硬盘 | `hdd` | 2 | /hard_drives/ |

## 3. 列表条目解析规则（第一层）

### 3.1 条目定位与标识

- 条目根：`<div class="pitem clearfix s_{proId} p_{proId}" reldate="{proId}" relmanu="{manuId}">`
- `pro_id`：正则 `p_(\d+)` 或 `reldate="(\d+)"`
- `manu_id`：`relmanu="(\d+)"`
- 型号名：`.pro-intro h3 a` 的 `title` 属性
- 深参链接：`.morep` 的 `href`（**直接取，不自算前缀**）
- 详情链接：`h3 a` href
- 热度：`h3` 内 `本月<i>(\d+)</i>人已选用`

### 3.2 价格（三个来源，优先级明确）

| 字段 | 选择器 | 说明 |
|------|--------|------|
| `zol_show` | `.price-box .price`（￥1359） | 攒机页主展示价 |
| `zol_min` | `.other .sjtotal`（N商家:￥675(最低价)） | 多商家最低价 |
| `zol_jd` | `.other .jd`（京东:￥769） | 京东价（**兜底主基准，见价格设计**） |

正则：`价格数字 = re.search(r'￥\s*([\d,]+)', text)`，去逗号转 int。

### 3.3 内嵌参数解析

```python
# .paramet 内每个 span：<span title="..">标签：<em>值</em></span>
params = dict(re.findall(r'<span[^>]*>([^：<]+)：<em>(.*?)</em></span>', paramet_html, re.S))
# 值去首尾空白；注意值可能被截断（见 §4）
```

各品类内嵌 5 字段（实测确认）：

| category | 内嵌 5 字段 |
|----------|-------------|
| cpu | 插槽类型 / CPU主频 / 加速频率 / 核心数量 / 制作工艺 |
| mainboard | 主芯片组 / CPU插槽 / 主板板型 / 集成芯片 / 内存类型 |
| memory | 容量描述 / 内存类型 / 内存主频 / CL延迟 / 适用类型 |
| gpu | 芯片厂商 / 显卡芯片 / 显存容量 / 显存位宽 / I/O接口 |
| hdd | 硬盘容量 / 接口类型 / 转速 / 缓存 / 硬盘尺寸 |
| ssd | 存储容量 / 接口类型 / 读取速度 / 缓存 / 外形尺寸 |
| psu | 额定功率 / PFC类型 / 80PLUS认证 / 主板接口 / 电源模组 |
| cooler | 散热器类型 / 散热方式 / 适用范围 / 最大风量 / 轴承类型 |
| case | 机箱类型 / 机箱结构 / USB接口 / 3.5英寸仓位 / 机箱材质 |

## 4. 深参页解析规则（第二层，兼容校验必需）

### 4.1 为何必需

列表内嵌是**精简截断版**，兼容关键字被切：散热器「适用范围」→「Intel：LGA 115X/」（截断）、固态「外形尺寸」→「80.15×22.15×2.」、机箱「USB接口」含 `<br>` 截断。这些是兼容校验硬依据，**必须进深参页补全**。

### 4.2 URL 与解析

```
URL: 从列表 .morep href 直接取（形如 //detail.zol.com.cn/1404/1403088/param.shtml）
GBK 解码。表格结构：
  <th><span id="newPmName_{N}">标签</span></th>
  <td><span id="newPmVal_{N}">值</span></td>
```

```python
names = dict(re.findall(r'id="newPmName_(\d+)"[^>]*>(.*?)</span>', html, re.S))
vals  = dict(re.findall(r'id="newPmVal_(\d+)"[^>]*>(.*?)</span>', html, re.S))
deep = {names[k]: _clean(vals[k]) for k in names if k in vals}
# _clean：去 HTML 标签（<br>→ / ）、去首尾空白
```

### 4.3 各品类必抓深参字段（补 compat 用）

| category | 深参必取字段 | 归一目标（compat 键） |
|----------|-------------|----------------------|
| cpu | 插槽类型 / 热设计功耗(TDP) / 内存类型 | socket / tdp_w / mem_type |
| mainboard | CPU插槽 / 主板板型 / 内存类型 / 内存插槽(数量) | socket / form_factor / mem_type |
| memory | 内存类型 / 容量 / 频率 | mem_type / capacity_gb |
| gpu | 芯片 / 显存容量 / **显卡长度 / 电源接口 / 建议电源** | vram_gb / length_mm / power_pin / rec_psu_w / tdp_w |
| ssd | 接口类型 / **外形尺寸(M.2 2280等)** / 容量 | interface / form_factor / capacity_gb |
| hdd | 接口类型 / 尺寸 / 容量 | interface / size_inch / capacity_gb |
| psu | 额定功率 / 模组 / 80PLUS / **尺寸** | rated_w / modular / cert |
| cooler | **适用范围(完整插槽列表)** / 散热方式 / **散热器高度** | cpu_socket_support[] / type / height_mm |
| case | **机箱结构(支持板型)** / **最大显卡长度 / 最大散热器高度** | form_factors[] / max_gpu_len_mm / max_cooler_h_mm |

> 深参字段名以实测为准；若某型号缺字段，compat 对应键置 null，兼容校验按「未知→放行但告警」处理（见 04 兼容引擎设计）。

## 5. 天梯图解析规则（D5 性能分地基）

### 5.1 CPU 天梯（JSON 接口）

```
GET https://cpu.zol.com.cn/router.php?c=Tianti_Cpu&a=GetList
Referer: https://cpu.zol.com.cn/soc/    GBK 解码
返回: {"colligate":[...], "singleCore":[...], "multiCore":[...], "game":[...]}
每项: {"firm":"2","proId":"2118101","model":"AMD Ryzen 9 9950X3D（游戏）","score":"409.0","rank":1,"ratio":"99.3%"}
```

- **装机场景取 `game` 维度**（游戏性能分），`colligate`（综合）作备用。
- `proId` 与 GetGoods 的 `pro_id` **直接对齐**。
- `score` 转 float 作 `tier_score`；`firm`：1=Intel、2=AMD。

### 5.2 显卡天梯（静态 HTML）

```
GET https://vga.zol.com.cn/soc/    Referer: https://vga.zol.com.cn/    GBK 解码
结构: <div class="item-box clearfix" data-proid="{proId}"> ... <div class="num">{score}</div>
      型号在 <dt data-index="0">{model}</dt>；排名/领先在 .article h4 span
```

- `data-proid` 与 GetGoods `pro_id` 对齐。
- `.num` 文本转 int 作 `tier_score`（显卡分值域与 CPU 不同，各自独立，不跨类比较）。

### 5.3 天梯对齐策略

天梯型号名可能带后缀（「（游戏）」「D Vulcan OC」等），与攒机库不一定字面相同，**但 `proId` 相同**——优先用 `proId` join，join 不上再退型号名模糊匹配（见 03 价格/对齐设计的匹配算法，复用同一套归一）。

## 6. 抓取工程规范

### 6.1 反爬与礼貌

| 项 | 规则 |
|----|------|
| Referer | GetGoods→`zj.zol.com.cn/`；天梯→对应频道页 |
| 并发 | ≤ 2 |
| 间隔 | 每请求 ≥ 1s，品类间/分页间加 0.5~1.5s 随机抖动 |
| 编码 | 全部 GBK→UTF-8 |
| 重试 | 单请求失败退避重试 3 次（1s/3s/5s），仍失败记录并跳过 |
| robots | P2 实施前复核 `zol.com.cn/robots.txt`，仅取公开规格/价格 |

### 6.2 抓取流程（每次全量刷新）

```
for category, subcate_id in 9品类:
    page 1 → 取 maxPage
    for page in 1..maxPage:
        GetGoods → 解析条目列表（第一层字段 + 内嵌参数 + 三价 + 热度 + morep url）
        for 每条:
            if 需补 compat（首次入库 或 compat 缺字段）:
                抓 param.shtml → 合并深参字段
    天梯: 抓 CPU JSON + 显卡 HTML → 更新 perf_tier
落库（见 §7 幂等写入）
```

### 6.3 深参抓取优化（控制量）

深参页数量大（如内存 4050 条），全量抓不现实。策略：
- **只对「可能进方案」的件抓深参**：按热度/价格合理区间过滤（如各品类 Top-N 热度 + 覆盖主流价格档）。
- 已抓深参的 `pro_id` 带 `deep_fetched_at`，长缓存（规格定型后不变），刷新时跳过已抓且非新品的。
- N 值可配（`DEEP_TOP_N`，默认 300/品类），保证方案候选池够用又不过量抓取。

## 7. 落库（幂等）

- 写入前**备份 `.db`**（SSOT §7 铁律）。
- 以 `pro_id` UPSERT：存在则更新 specs/compat/price/popularity/fetched_at，不存在则插入。
- 停售件（本轮 GetGoods 未出现）标 `active=0`，不物理删除（保留历史/供已生成方案追溯）。
- 天梯 `perf_tier` 以 `pro_id` UPSERT。
- 一次刷新在**事务**内提交，失败回滚，不留半量库。

## 8. 触发方式（安全，见审计-3）

- 爬虫**不作为 MCP 工具暴露**给 claude。
- 由容器内 **cron / 独立入口脚本** 触发（`CRAWL_CRON`，默认每日 04:00）。
- 支持手动触发：容器内 `python -m app.crawler.run --category all|<cat>`（运维用，非 MCP 面）。

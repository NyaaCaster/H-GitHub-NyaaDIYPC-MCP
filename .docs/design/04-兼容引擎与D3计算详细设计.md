# 04 兼容引擎与 D3 计算详细设计

> 对应 SSOT D3（计算铁律）+ 第 4 节（兼容硬约束）+ P4。
> **D3 铁律：所有数值判定（预算求和、功耗估算、in_budget、兼容布尔）全部由服务端 Python 计算，claude/LLM 永不做算术。** 本文件把每条规则写成可直接编码的判定式。

---

## 1. D3 计算铁律（不可违背）

| 计算项 | 由谁算 | 说明 |
|--------|:---:|------|
| 方案总价 `total = Σ effective_price` | Python | LLM 禁止求和 |
| 预算判定 `in_budget = budget_min ≤ total ≤ budget_max` | Python | 闭区间 |
| 功耗估算 `est_power` | Python | 见 §3 |
| 电源余量 | Python | 见 §3 |
| 兼容布尔 | Python | 见 §2 |
| 天梯达标判定 | Python | 见 05 |

claude 只做：理解需求语义、挑选候选倾向、对 Python 产出的方案做**文字性**复核与解释。任何数字以 Python 返回为准。

## 2. 兼容校验规则（validate_build 核心）

输入：一组件 `{cpu, mainboard, memory, gpu?, ssd?, hdd?, psu, cooler, case}`。逐规则判定，输出 `{compat_ok: bool, issues: [{level, rule, detail}]}`。`level ∈ {error, warn}`。error 使 compat_ok=False；warn 不阻断但上报。

### 2.1 硬规则（error 级，任一违反即不兼容）

| 编号 | 规则 | 判定式 |
|------|------|--------|
| C1 | CPU↔主板插槽 | `cpu.socket == mainboard.socket` |
| C2 | 内存类型↔主板 | `memory.mem_type == mainboard.mem_type` |
| C3 | 内存类型↔CPU | `memory.mem_type == cpu.mem_type`（CPU 未标则跳过） |
| C4 | 主板板型↔机箱 | `mainboard.form_factor ∈ case.case_ff[]` |
| C5 | 显卡长度↔机箱 | `gpu.gpu_len_mm ≤ case.max_gpu_len_mm` |
| C6 | 散热器高度↔机箱 | `cooler.cooler_h_mm ≤ case.max_cooler_h_mm` |
| C7 | 散热器↔CPU 插槽 | `cpu.socket ∈ cooler.cooler_sockets[]` |
| C8 | 电源额定↔需求 | `psu.rated_w ≥ est_power * 1.3`（见 §3，余量硬约束） |
| C9 | 存储接口↔主板 | M.2 NVMe 件数 ≤ `mainboard.m2_slots`（无 M.2 槽则 SATA 兜底） |

### 2.2 软规则（warn 级，工程软约束，审计整改 3）

| 编号 | 规则 | 判定 |
|------|------|------|
| W1 | K 系 CPU 配 Z 板 | Intel 型号含 K 且 `mb_chipset` 非 Z 开头 → warn「超频需 Z 板」 |
| W2 | 高端 CPU 建议规格内存 | 略（提示 DDR5 频率匹配） |
| W3 | 供电余量偏紧 | `1.3 ≤ psu.rated_w/est_power < 1.5` → warn（够用但不宽裕，见 §3 目标 1.5） |
| W4 | 尺寸余量偏紧 | 显卡/散热器与机箱上限差 < 10mm → warn |
| W5 | 无独显且 CPU 无核显 | `gpu is None and not cpu.igpu` → warn「无图形输出」 |

### 2.3 NULL 处理

任一参与字段为 NULL（深参缺失）→ 该规则记 `warn`（rule=Cx_unknown, detail「字段缺失无法校验」），**不判 error**（放行但告警），避免因数据缺口误杀合理搭配。

## 3. 功耗估算与电源余量（§2 C8/W3 依据）

### 3.1 估算式（Python）

```
est_power = cpu.tdp_w * CPU_FACTOR      # CPU_FACTOR=1.3（峰值/瞬态余量）
          + (gpu.tdp_w or gpu 建议电源反推) * 1.0
          + BASE_LOAD                    # 主板+内存+存储+风扇 基础负载，默认 80W
```

- GPU 功耗优先 `gpu.tdp_w`；缺则用 `gpu_rec_psu_w` 反推整机（`rec_psu_w` 本就是整机建议，作 est_power 下限参考）。
- 常量 `CPU_FACTOR / BASE_LOAD` 落 env（§5 环境变量），默认写死可跑。

### 3.2 电源选择目标

```
硬约束（C8）: psu.rated_w ≥ est_power * 1.3
推荐目标   : psu.rated_w ≥ est_power * 1.5   # 搭配算法优先选满足 1.5 的电源
```

## 4. validate_build 工具行为

- 输入：件 `pro_id` 列表（或完整件对象）。
- 服务端按 pro_id 查 compat/hardware，跑 §2 全规则 + §3 功耗，算 total。
- 输出：
```json
{
  "compat_ok": true,
  "total": 9280,
  "est_power": 520,
  "psu_headroom": 1.44,
  "issues": [{"level":"warn","rule":"W3","detail":"供电余量偏紧(1.44<1.5)"}]
}
```
- **只读工具**（审计整改 1）：validate_build 不写库、不改状态。

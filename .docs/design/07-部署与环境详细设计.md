## 4. 构建发布（rebuild.py，Windows）

标准流程（见工作空间 CLAUDE.md）：
```
1. 读 .env → PRIVATE_DOCKER_REGISTRY_HOST（本地构建用 localhost:5000）
2. git rev-parse --short=7 HEAD → tag
3. docker build → ${HOST}/nyaadiypc-mcp:<sha> + :latest
4. docker push（失败重试 3）
5. 注册表清理旧清单
6. 本地清理旧标签 + 悬空镜像
7. 输出掩码仓库地址为 <PRIVATE_REGISTRY>
```
> rebuild.py 只构建推送，**不启动容器**（工作空间铁律）。

## 5. macmini 部署（restart.py）

```
1. mkdir -p /root/DockerContainer/NyaaDIYPC-MCP/ + DockerRes/NyaaDIYPC-MCP/db/
2. 传 .env（真实仓库地址 192.168.31.142:5000）+ docker-compose.publish.yml(→docker-compose.yml) + restart.py
3. docker compose pull → down → up -d → image prune -f → ps
```
- macmini docker 经 snap，SSH 命令加 `export PATH=$PATH:/snap/bin &&`。
- `insecure-registries` 已含 192.168.31.142:5000。
- **首次上线需先跑一次爬虫填库**：容器内 `python -m app.crawler.run --category all`，再验证 MCP 工具。

## 6. 安全红线（汇总）

1. 私有仓地址仅经 .env 注入，git 文件掩码 `<PRIVATE_REGISTRY>`。
2. MCP 只读工具面，写操作不暴露（审计整改 1）。
3. DB 写前备份（SSOT §7）。
4. 未经用户明确要求不 `up`/`run`/`start` 容器；本设计涉及的 up/爬虫首灌均需用户明确许可后执行。
5. 出站仅访问 ZOL（抓取）+ best-price MCP（补价），不外传代码/数据。
6. best-price 补价强制过滤层（03 §2.2）防污染。
